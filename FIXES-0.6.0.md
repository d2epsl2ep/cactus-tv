# v0.6.0 播放器修复说明

本版本针对 v0.5.1 播放器审计中发现的问题做了完整修复，不改变 Cloudflare Pages + D1 的部署方式，也不新增数据库迁移。

## 启动与流畅度

- 详情页空闲时预载播放器模块和 HLS 引擎。
- HLS 启用首分片预取；下一集在临近结束或当前源稳定后预热播放清单。
- 播放启动以首帧为准，不再把仅拿到 metadata/manifest 误判为成功。
- 根据省流量、网络类型、下行速度、RTT、内存和 CPU 自动调整初始码率与缓冲区。

## 容灾

- 新增等待/停滞监控，播放进度长时间不增长时先恢复引擎，仍失败再触发换源。
- 自动换源最多尝试 7 次，总预算 48 秒，避免坏源逐个长时间挂起。
- 先比较不同数据源的最佳线路，再测试剩余线路，降低整个数据源故障时的等待。
- 片源评分增加时间衰减，并细分到媒体域名和线路，不再被很久以前的累计成功次数长期绑架。

## 格式与代理

- 新增 DASH 播放、清晰度、多音轨和内嵌字幕支持。
- 代理与直连均可探测没有 `.m3u8`/`.mpd` 后缀的清单。
- 修复全局中间件覆盖 `/api/stream` 缓存头的问题。
- HLS 清单重写分片/密钥；DASH 修正 BaseURL 并通过 RequestModifier 代理后续分片。
- 媒体响应保留 Range、ETag 和分片缓存。

## 字幕、设备与维护

- 远程字幕改走受控字幕代理，限制大小、超时、类型和重定向，并识别 UTF-8、UTF-16、GB18030、Big5。
- 修复换集、换源和退出后的旧字幕残留。
- 手机横屏恢复关闭、字幕、音轨、换源入口；窄屏保留上一集/下一集。
- 增强电视遥控器焦点导航、返回键和媒体键。
- 增加播放诊断面板，显示引擎、分辨率、码率、缓冲、带宽、首帧、丢帧、卡顿和媒体域名。
- 增加 ES2016 `nomodule` 兼容包与 dialog 降级。
- 管理密钥最低长度统一为 8 个字符。
- 删除根目录重复的 `app.js/player.js/player-ui.js`，只保留 `public/js/` 运行版本。
