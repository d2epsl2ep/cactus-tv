# Changelog

## 0.6.0

- 首次播放改为首帧确认，稳定播放后才记录片源成功。
- 新增卡死检测、分级重连、自动换源时间预算与尝试上限。
- HLS 引擎提前预热，启用首分片预取与下一集播放列表预热。
- 代理端可识别无 `.m3u8` 后缀的 HLS，并保留播放列表/分片缓存策略。
- 新增 DASH、HLS 多音轨、内嵌字幕与播放诊断面板。
- 远程字幕改走受控代理，增加超时、大小限制和常见中文编码识别。
- 修复切集/换源后的字幕残留。
- 手机横屏恢复关闭、字幕和换源入口，小屏保留上一集/下一集。
- 增强电视遥控器焦点导航与媒体按键。
- 增加旧浏览器 `nomodule` 兼容包和 dialog 降级。
- `ADMIN_TOKEN` 最低长度统一为 8 个字符。
- 删除根目录重复的播放器脚本，运行代码统一位于 `public/js/`。

# 更新日志

## v0.5.1 Smart Preload

- 首页首帧仍保持轻量，但首帧后会主动准备后续 2～3 屏卡片。
- 横向片单图片观察范围扩大，用户滑动前即开始下载海报。
- 进入下一分类区域前会提前约 760～1180px 预渲染该行。
- 滑动过程中维持动态预加载窗口，不再等到行尾才补卡片。
- 开启系统省流量时自动收缩预加载范围，避免浪费流量。

## v0.5.0 Turbo

### 极致流畅度

- 首页由一次性渲染整页卡片改为首屏小批量渲染，接近视口或横向滚动时再追加。
- 搜索结果“显示更多”只追加新卡片，不再销毁并重建已经显示的节点。
- 图片改为 IntersectionObserver 预加载，低端设备使用更保守的预取距离。
- 长剧集列表按批次创建按钮，避免数百集同时占用 DOM 和事件资源。
- 自动识别省流量、低内存、低核心数和减少动态效果设备，关闭高耗 GPU 模糊、阴影和动画。
- 平板横屏海报取消每张卡片一层实时模糊背景，保留完整人像且降低合成开销。

### 加载速度

- 首页数据与站点健康检查并行加载，不再串行等待。
- 播放器界面和播放引擎改为真正按需加载，首页不解析播放器模块和 hls.js。
- 首屏脚本使用 modulepreload，首页、详情和搜索的浏览器缓存时间延长。
- 豆瓣海报优先请求适合卡片的中等尺寸，并直接尝试列表海报；资料页查询只在原图失效时回退。
- 海报浏览器缓存提高到 1 天，Cloudflare 边缘缓存提高到 7 天，并启用长期 stale-while-revalidate。
- 播放器时间轴更新合并到 requestAnimationFrame，减少频繁布局与绘制。
- 低端设备缩小 HLS 前后缓冲区，降低内存峰值和后台解码压力。

### 管理后台

- `ADMIN_TOKEN` 最低长度由 16 个字符调整为 8 个字符，前端和服务端规则保持一致。
- 仍建议使用随机字母、数字和符号组合；8 位只代表最低可用长度。

### 部署

- 不新增数据库迁移、Binding 或环境变量。
- 覆盖全部文件并提交到原 GitHub 仓库即可自动部署。

## v0.4.0

### 播放与容灾

- 播放失败后自动尝试同一数据源的其他线路。
- 同源线路全部失败后，按接口延迟和本机历史成功率尝试其他数据源。
- 自动换源时保留当前播放进度、集数和字幕入口。
- 播放器增加上一集、下一集和手动切换备用线路。
- 播放结束后显示 5 秒倒计时并自动进入下一集，可在设置中关闭。
- 当前使用的数据源和线路会显示在播放器下方。

### 页面与路由

- 增加首页分类入口。
- 搜索结果增加类型、年份和片源筛选。
- 搜索结果每次先渲染 24 条，可继续加载，降低手机和平板的瞬时渲染压力。
- 增加 `/detail/...`、`/watch/...`、`/category/...`、`/search`、`/favorites` 和 `/history` 直达路由。
- 刷新播放页或详情页后可以恢复对应页面，不再只能回到首页。

### 性能

- 搜索 API 使用 Cloudflare Cache API 缓存结果。
- 上游数据源搜索限制为最多 5 路并发，避免大量数据源同时请求造成抖动。
- 浏览器搜索缓存由 30 秒延长至 2 分钟。
- 继续保留请求去重、超时取消、图片懒加载和播放器资源释放。

### 部署

- 不新增数据库迁移。
- 不新增 Cloudflare Binding 或环境变量。
- 已连接 GitHub 的 Cloudflare Pages 项目只需提交代码，等待自动部署。
